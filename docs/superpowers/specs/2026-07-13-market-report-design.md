# 채용 시장 분석 HTML 리포트 — `08 시장 리포트 (Market Report)`

## 배경 / 목적
DB에 쌓인 375개 공고 데이터로 **채용 시장 인사이트**(무엇이 수요가 많고, 어떤 경력을 원하는지)를 한눈에 보는 자체완결 HTML 리포트를 n8n 워크플로우로 자동 생성한다. 데이터 다루는 역량을 보여주는 포트폴리오 요소.

## 확정된 결정
- **초점**: 채용 시장 분석(공고 데이터). 지원 퍼널/합격률은 실데이터가 없어 제외(추후 채워지면 섹션 추가).
- **생성**: n8n 워크플로우(수동 실행, 스케줄 옵션). DB 집계 → HTML 문자열 조립 → data/output 저장.
- **차트**: 외부 라이브러리 없이 **순수 CSS 바 차트**(자체완결, 오프라인·어디서든 렌더).

## 데이터 현실 (충실도)
- done 공고 375, tech_stacks 370, min_career 375 → 충실.
- locations 33, closed_at 33 → 희박 → 리포트에서 제외.
- 수집 시계열 거의 하루치 → 시계열 차트 제외.

## 아키텍처 (n8n)
```
수동 실행(Manual, +스케줄 옵션)
  → 요약(Postgres, executeOnce)        총건수·소스별·평균경력·기술종류수·수집기간
  → 기술 TOP(Postgres, executeOnce)     unnest(tech_stacks) TOP 15
  → 경력 분포(Postgres, executeOnce)     신입/주니어(1-2)/미들(3-5)/시니어(6+)
  → 소스별(Postgres, executeOnce)        source별 건수·평균경력
  → HTML 생성(Code)                      $('노드').all()로 집계 참조 → 자체완결 HTML 조립
  → 파일 저장(readWriteFile)             /data/output/market-report.html
```
- 각 Postgres 노드는 **executeOnce=true** — 입력 아이템 수만큼 쿼리가 반복 실행되는 것 방지(집계는 1회만).
- HTML 생성 Code는 읽기전용/RW 상관없이 조회만 하지만, 04처럼 **읽기전용 롤(jobs_ro)** 사용.

## 리포트 섹션
1. **헤더** — 제목 · 생성시각 · 기준(공고 N건, 수집기간)
2. **요약 카드 4개** — 총 공고 / 소스 분포 / 평균 요구경력 / 기술 종류 수
3. **기술스택 수요 TOP 15** — 수평 CSS 바 차트(핵심)
4. **경력 분포** — 4구간 바
5. **소스별 비교** — 원티드 vs 점프잇(건수·평균경력)

## 스타일
- 카드 레이아웃 + CSS 그라디언트 바, 라이트/다크(prefers-color-scheme) 대응, 반응형.
- 완전 자체완결: 인라인 CSS, 외부 이미지·CDN·JS 없음. HTML 이스케이프로 XSS 방지.

## 전달
- data/output/market-report.html 저장 → 브라우저에서 열람(생성시각은 헤더에 표기).
- 파일명 고정(최신본). 스케줄 추가 시 매일 수집 후 갱신.

## 에러 처리
- 집계 쿼리 실패 시 해당 섹션 비움(전체는 렌더). 데이터 0건이어도 "데이터 없음" 표시.

## 검증
1. 집계 쿼리 4종 결과 정확성(psql).
2. 생성 HTML을 브라우저에서 렌더 확인(레이아웃·바 비율·라이트/다크).

## 향후

- 지원 데이터 쌓이면 "지원 현황·합격률" 섹션 추가.
